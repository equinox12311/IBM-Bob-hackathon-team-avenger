### 4. Slash Commands (bob/commands/)
Bob created three slash commands:

**diary-save.md:**
- Explicit save with secret detection
- Validates input before MCP call
- Supports tags and code locations
- Returns entry ID on success

**diary-recall.md:**
- Semantic search interface
- Configurable k parameter
- Returns formatted results with citations
- Handles empty results gracefully

**diary-timeline.md:**
- Recent entries view
- Configurable limit
- Reverse-chronological order
- Formatted with timestamps

### 5. Mode Rules (bob/rules-cortex/)
Bob generated four mode rules:

**01-capture-style.md:**
- Entry formatting guidelines
- Specific over generic
- Include context and reasoning
- Use tags appropriately

**02-no-secrets.md:**
- Secret detection enforcement
- Refuse to save API keys, tokens, passwords
- Suggest redaction strategies
- Security-first approach

**03-proactive-recall.md (Innovation #1):**
- Auto-trigger on file open/focus
- Compose query from file path + class/function names
- Surface entries with score > 0.5
- Silent if no matches
- Show top 3 with file:line citations

**04-agentic-auto-capture.md (Innovation #2):**
- Watch for task completion signals
- Compose one-sentence learning summary
- Propose diary_save with confirmation
- Never silent - always ask first
- Captures "why" not just "what"

## Innovation Highlights

### Proactive Recall
Unlike traditional tools that wait for user queries, Cortex automatically surfaces relevant past learnings when Bob opens a file. This changes the workflow from "remember to search" to "automatically reminded."

### Agentic Auto-Capture
Instead of requiring manual journaling, Bob proposes draft entries when tasks complete. The developer just confirms, making capture frictionless.
