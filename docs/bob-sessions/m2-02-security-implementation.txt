## Task Description
Implement comprehensive security features for Cortex API including secret detection, rate limiting, and security headers.

## Bob's Contributions

### 1. Secret Detection System (cortex_api/secrets.py)
Bob generated a robust secret detection module with 11 pattern types:
- AWS access keys and secret keys
- GitHub personal access tokens
- Slack tokens and webhooks
- OpenAI API keys
- Stripe API keys
- JWT tokens
- Private keys (RSA, SSH, PGP)
- Database connection strings
- Generic high-entropy secrets

Implementation includes:
- Regex pattern matching for known secret formats
- Shannon entropy calculation for generic secrets
- Redacted findings in error responses
- HTTP 422 status for rejected entries

### 2. Rate Limiting (cortex_api/server.py)
Bob implemented in-memory sliding window rate limiting:
- 60 requests/minute per IP (default)
- 20 creates/minute for POST endpoints
- 30 searches/minute for search endpoints
- 10 LLM calls/minute for generation endpoints
- Configurable via environment variables

### 3. Security Headers Middleware
Bob added OWASP-recommended security headers:
- X-Frame-Options: DENY
- X-Content-Type-Options: nosniff
- Referrer-Policy: strict-origin-when-cross-origin
- Permissions-Policy: restrictive defaults
- HSTS (production only)
- Server fingerprint removal

### 4. Authentication System (cortex_api/auth.py)
Bob implemented bearer token authentication:
- Token validation on all REST endpoints (except /health)
- Secure token comparison
- Clear error messages for unauthorized access
- MCP transport inherits OS user security
