## 1 · Get a working stack in 5 minutes

```bash
git clone https://github.com/equinox12311/IBM-Bob.git
cd IBM-Bob
make setup          # venv + Python deps + npm + .env (~3 min first run)
make install-bob    # writes ~/.bob/settings/mcp_settings.json + workspace .bob/mcp.json
make dev            # api on :8080, web on :5173
```

Open **http://localhost:5173** → token `test`.

In **IBM Bob** → reload window (`⌘+Shift+P → Developer: Reload Window`) → switch to the **📓 Cortex** mode → try `/diary-save anything`.
The entry will appear immediately in the web UI's Timeline — same SQLite, two transports.

If the MCP server doesn't connect:
```bash
cat ~/.bob/settings/mcp_settings.json   # should contain mcpServers.cortex
make verify-mcp                          # opens MCP Inspector — verifies tools without burning Bob coins
```

## 2 · Bob session protocol (every member, every session)

The hackathon rules require **all relevant Bob sessions exported**. Read this **before** opening Bob.

For each session you run:

1. **Note the current commit** before you start.
2. **Use Bob deliberately** — clear, focused prompt. One task, one session.
3. **Export the task-session report** from Bob → save to `docs/bob-sessions/<member>-<NN>-<topic>.txt`
   (e.g. `m2-01-pytest-edges.txt`).
4. **Commit** with this format:
   ```
   feat: <description>

   Generated by IBM Bob (M<X>) — session #<N>
   Report: docs/bob-sessions/<member>-<NN>-<topic>.txt
   Co-Authored-By: IBM Bob <bob@ibm.com>
   ```
5. **Append a row** to `docs/BOB_USAGE.md`:
   ```
   | <N> | <YYYY-MM-DD> | M<X> | P<phase> | <task> | <coins> | <commit> | bob-sessions/<file> |
   ```

**Skipping any of these = the session doesn't count for the submission.** Set a 5-min "wrap-up" reminder when you start a session.

## 7 · If you get stuck

- **MCP server won't connect in Bob** → re-run `make install-bob`, reload Bob window, check `~/.bob/settings/mcp_settings.json`
- **Web UI shows a blank page** → check the API is up (`curl http://localhost:8080/health`), check the bearer token in Settings
- **`pip install` fails on llama-cpp-python** → it's optional; skip with `LLM_PROVIDER=off` in `.env`
- **sqlite-vec warning** → harmless; we fall back to Python cosine. Use Homebrew Python to silence it.

Anything else: post in `#cortex-hackathon` and tag M1.
