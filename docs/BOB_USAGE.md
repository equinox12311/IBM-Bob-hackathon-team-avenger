# IBM Bob Usage Ledger

This file tracks **every** IBM Bob task-session run by every team member during the hackathon (May 1–3, 2026). The submission rules require an exported report of *all relevant tasks/sessions used for the contest* — this ledger is the index.

## How to add a row (every Bob session, every member)

1. **Run** the Bob session deliberately on a focused task
2. **Export** the task-session report from Bob (Bob → Export Session) → save to `docs/bob-sessions/{your-id}-{nn}-{topic}.txt` (e.g. `m2-03-pytest.txt`)
3. **Commit** with the format:
    ```
    feat: {description}

    Generated by IBM Bob (M{X}) — session #{N}
    Report: docs/bob-sessions/{file}
    Co-Authored-By: IBM Bob <bob@ibm.com>
    ```
4. **Append** a row to the table below
5. **Update** your row in the per-member coin-spend totals

## What to use Bob for (high-leverage tasks)

- Refactoring core logic into clean modules
- Generating test suites (pytest, vitest)
- Security / secret scanning
- Generating documentation (README sections, API docs)
- Generating Bob extension manifests (mode/skill/rules — Bob building Bob)
- The live demo session (recorded into demo.mp4)

## What NOT to use Bob for (use Claude Code/Cursor instead)

- Initial scaffolding (FastAPI app, Vite project)
- Boilerplate config files
- Exploratory "what does X look like" sessions
- Bug-bashing iterations
- Anything where Bob's coin cost > value-per-session
