## Cortex-Specific Implementation

### IBM Bob Integration (All 5 Layers)

1. **MCP Server** - 5 tools exposed via stdio protocol
   - `diary_save` - Save journal entries
   - `diary_recall` - Semantic search
   - `diary_timeline` - Recent entries
   - `diary_link_code` - Attach code locations
   - `diary_feedback` - Boost/flag entries

2. **Custom Mode** - `📓 Cortex` mode
   - Slug: `cortex`
   - Auto-loads skill and rules
   - Includes MCP tool group
   - Oriented around proactive recall + agentic capture

3. **Skill** - Playbook at `.bob/skills/cortex/`
   - When to capture entries
   - When to recall entries
   - How to format entries
   - Feedback loop guidelines

4. **Slash Commands** - Three commands
   - `/diary-save` - Explicit save with secret detection
   - `/diary-recall` - Semantic search
   - `/diary-timeline` - Recent entries

5. **Mode Rules** - Four rules in `.bob/rules-cortex/`
   - `01-capture-style.md` - Entry formatting
   - `02-no-secrets.md` - Secret refusal
   - `03-proactive-recall.md` - Auto-surface on file-open
   - `04-agentic-auto-capture.md` - Propose saves on completion

### Key Innovations

1. **Proactive Recall** - Bob auto-surfaces related entries when opening files (score > 0.5)
2. **Agentic Auto-Capture** - Bob proposes saves on task completion (one-click confirm)
3. **Five-Layer Integration** - Uses ALL Bob extension surfaces
4. **Learning Loop** - Feedback signals + recency decay evolve the diary

## Installation for Judges

### Quick Start (One-Click)
```bash
git clone https://github.com/equinox12311/IBM-Bob.git
cd IBM-Bob
make setup     # creates venv, installs Python + npm deps, copies .env
make dev       # starts api on :8080 and web on :5173
```

Then open **http://localhost:5173** and paste `test` as the bearer token.

### For Judges - Full Bob Integration
```bash
make judge     # = make setup + make install-bob (copies extensions + patches MCP config)
make dev       # then start the stack
```

`make install-bob` is idempotent: it copies the mode, skill, slash commands, and four mode rules into `~/.bob/`, and registers the `cortex` MCP server in `~/.bob/settings.json` with absolute paths derived from your venv.

Restart Bob, switch to the **📓 Cortex** mode, and `/diary-save` will land in the same SQLite the web UI is reading from.

### Alternative: Docker
```bash
docker compose up --build      # api:8080, web:8081, bot
# or
make up
```

## Verification Without Bob
